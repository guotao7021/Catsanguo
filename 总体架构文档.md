# 猫三国 - 总体架构文档

> 最后更新：2026-04-15（根据需求文档同步）

## 一、总体架构目标

- 支持三国志11式事件剧本开局（选剧本→选势力）
- 支持回合制时间推进（10日/月/季多层迭代）
- 支持单机策略游戏完整体验
- 支持多武将编队战斗（当前3人编队）+ 武将俘获机制
- 支持外交系统（同盟/通商/停战/劝降）
- 支持内政官员分配制（所有操作需武将执行）
- 支持技能 / Buff / AI 复杂扩展
- 支持数据驱动内容生产（JSON配置）
- 可持续迭代

## 二、技术选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 游戏框架 | MonoGame (XNA) | C# 跨平台2D游戏框架 |
| 渲染方式 | SpriteBatch | 2D精灵渲染，预乘Alpha |
| 字体系统 | FontStashSharp | 支持TTF动态字体 |
| 数据格式 | JSON | System.Text.Json 序列化 |
| 语言 | C# 8.0+ | .NET平台 |

### 3.1 表现层（Presentation）

**职责**：窗口管理、渲染、输入处理

| 组件 | 类 | 职责 |
|------|-----|------|
| 游戏主类 | `CatSanGuoGame` | MonoGame主循环、资源加载、场景调度 |
| 渲染系统 | `SpriteBatch` | 2D精灵批量渲染 |
| 相机系统 | `Camera2D` | 世界/屏幕坐标转换、缩放、平移、边界约束 |
| 输入系统 | `InputManager` | 键鼠状态跟踪（点击/拖拽/滚轮） |
| 精灵管理 | `SpriteSheetManager` | 精灵表加载和管理 |

**Camera2D核心能力**：
```csharp
public Matrix GetTransformMatrix()       // 渲染矩阵
public Vector2 ScreenToWorld(Vector2)    // 屏幕→世界坐标
public Vector2 WorldToScreen(Vector2)    // 世界→屏幕坐标
public void ClampPosition()              // 边界约束（含HUD偏移）

```

### 3.2 场景层（Scene System）

**职责**：游戏流程管理，每个场景独立管理生命周期

```csharp
abstract class Scene {
    virtual void Enter()                 // 进入场景
    virtual void Update(GameTime)        // 逻辑更新
    virtual void Draw(GameTime)          // 渲染
    virtual void Exit()                  // 退出场景
}
```

| 场景 | 职责 |
|------|------|
| `MainMenuScene` | 主菜单、剧本选择、势力选择、开始游戏 |
| `WorldMapScene` | 世界地图探索、城池管理、军队调度、回合推进、外交 |
| `BattleScene` | 战斗执行（部署→倒计时→战斗→结算+俘获判定） |

**SceneManager** 管理场景切换，支持淡入淡出过渡效果。

```csharp
    // 官员分配
    public void AssignOfficer(string cityId, OfficerRole role, string generalId)

    // 编队管理
    public IReadOnlyList<string> CurrentSquad { get; }
```

### 3.4 数据层（Data Layer）

```

├── SkillData (技能配置)
├── SkillTriggerData (技能触发链)
├── CityData (城池数据)

├── BondData (羁绊数据)

配置表：
├── ScenarioConfig (剧本配置 scenarios.json)
├── InteriorConfig (建筑列表/升级费用/产量计算)
├── UnitCounterConfig (10×10军种克制表)

├── FormationConfigTable (阵型配置)
```

**数据加载流程**：
```
JSON文件 (scenarios.json, cities.json, generals.json等)
     ↓

     ↓
DataSchemas 数据实例
     ↓

     ↓
GameState 运行时数据
     ↓

```

## 六、UI系统架构

采用**即时模式（Immediate Mode）UI**，直接使用SpriteBatch绘制：

- 没有使用第三方UI框架
- 所有UI元素通过DrawRectangle/DrawString直接绘制
- 碰撞检测：InputManager.IsMouseInRect()
- 弹窗位置：根据城池屏幕坐标动态计算，防止超出屏幕

## 八、主要流程

### 游戏启动
```
CatSanGuoGame.Initialize()
  → LoadContent()（字体/纹理/精灵表）
  → GameRoot初始化（加载JSON数据）
  → SceneManager.ChangeScene(MainMenuScene)
```

### 开局流程（新增）
```
MainMenuScene
  → 新建游戏
  → 选择剧本（ScenarioManager加载scenarios.json）
  → 选择势力
  → ScenarioManager.InitializeWorld() → GameState初始化
  → TurnManager.Initialize(startDate)
  → SceneManager.ChangeScene(WorldMapScene)
```

### 回合循环（替代原30秒Tick）
```
WorldMapScene.Update()
  → 玩家操作阶段（自由操作，无时间压力）
    → 城池管理/官员分配/外交/编队出征
  → 点击"结束回合"
  → TurnManager.AdvanceTurn()
    → 日期+10日
    → 搜索/外交/行军结算
    → 历史武将登录检查
    → （月末）经济迭代
    → （季末）兵力迭代
    → 胜利/失败检查
```

### 战斗流程
```
BattleScene.Enter()
  → 创建Squad实例（绑定6维属性/技能/动画）
  → 初始化EventBus/BuffSystem/SkillTriggerSystem

BattleScene.Update()
  → Deploy：玩家部署武将
  → Countdown：3秒倒计时
  → Fighting：自动战斗 + 玩家释放技能
  → Result：XP/技能点/城池占领 + 俘获判定
```

## 十、关键架构决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 引擎 | MonoGame而非Unity | 轻量、完全C#控制、无引擎依赖 |
| 渲染 | 2D SpriteBatch | 策略游戏不需要3D |
| UI | 即时模式绘制 | 简单直接、无框架依赖 |
| 状态管理 | 单例GameState | 全局唯一、便于存档 |
| 数据 | JSON配置 | 易于编辑和扩展 |
| 架构 | 场景制而非ECS | 适合当前规模、降低复杂度 |
| 时间系统 | 回合制而非实时 | 符合三国志11策略体验 |
| 内政 | 官员分配制 | 人才=核心资源，增加策略深度 |

## 十一、架构演进方向（Simulation内核升级）

> 以下为架构升级规划，目标是将战斗逻辑从BattleScene中剥离为独立Simulation模块，
> 以支持战斗回放、网络同步和更好的可测试性。

### 11.1 目标架构（六层）

```
┌──────────────────────────────┐
│ Presentation（表现层）         │ ← MonoGame 渲染/UI（只读数据+显示）
├──────────────────────────────┤
│ Application（应用层）          │ ← Scene / 流程控制 / 输入→Command
├──────────────────────────────┤
│ Domain（领域层）               │ ← 非战斗规则（城池/武将/资源/编队/外交/回合）
├──────────────────────────────┤
│ Simulation（模拟层）⭐核心      │ ← 战斗核心（纯计算，无渲染依赖）
├──────────────────────────────┤
│ Infrastructure（基础设施）     │ ← EventBus / Command / Tick
├──────────────────────────────┤
│ Data（数据层）                 │ ← JSON配置（剧本/武将/技能/城池等）
└──────────────────────────────┘
```

### 11.2 当前 vs 目标对比

| 项目 | 当前状态 | 目标状态 |

### 11.3 BattleSimulation 核心设计

独立的战斗模拟引擎（纯C#，无MonoGame依赖）：

```csharp
public class BattleSimulation
{
    World world;          // 单位/组件/系统
    CommandQueue commands; // 输入
    EventQueue events;    // 输出

    public void Init(BattleInitData data);
    public void AddCommand(Command cmd);
    public void Tick();
    public List<GameEvent> GetEvents();
    public BattleResult GetResult();  // 含俘获判定结果
}
```

**World内部结构：**
```csharp
public class World
{
    public int Frame;
    public List<Unit> Units;
    public CommandQueue CommandQueue;
    public EventQueue EventQueue;
    
    // 核心系统
    public CombatSystem CombatSystem;
    public SkillSystem SkillSystem;
    public BuffSystem BuffSystem;
    public AttributeSystem AttributeSystem;
    public AISystem AISystem;
    
    public Random Random; // 固定seed，确保确定性
}
```

**Tick执行顺序：**
```
Frame Tick：
 1. Command执行（玩家输入/AI命令）
 2. AI决策（每5帧，低频）
 3. Combat计算（伤害/移动）
 4. Buff更新（持续/过期）
 5. Attribute重算（dirty flag）
 6. Skill触发（事件驱动链）
 7. Event输出 → 表现层消费
```

### 11.4 BattleScene改造方式

```csharp
// 改造后的BattleScene
BattleSimulation sim;

void Update()
{
    sim.Tick();
    var events = sim.GetEvents();
    PlayEffects(events); // 转换为表现层效果
}
```

### 11.5 Command系统

```csharp
public struct Command
{
    public int Frame;
    public CommandType Type;    // CastSkill / ChangeTarget / ...
    public int ActorId;
    public int TargetId;
    public object Data;
}
```

**优势：**
- 战斗回放：记录Command序列 + 固定seed = 可复现
- 网络同步：同步Command = Lockstep
- 可测试：Simulation可脱离渲染独立运行

### 11.6 必须执行的重构任务

| 任务 | 优先级 | 说明 |
|------|--------|------|
| 战斗逻辑剥离 | 最高 | BattleScene → BattleSimulation |
| 引入Command系统 | 高 | 直接调用 → Command驱动 |
| GameState瘦身 | 中 | 删除逻辑 → 只保留数据 |
| Event统一出口 | 中 | 所有表现 = Event驱动 |
| 随机数确定性 | 高 | 使用固定seed Random |
