# 猫三国 - 内政系统设计文档

> 最后更新：2026-04-15（根据需求文档同步）

## 一、系统定位与目标

### 系统定位

内政系统 = 资源生产 + 城市发展 + 官员管理 + 外交 + 经济 + 军事后勤

### 核心目标
- 支撑战斗系统（兵力 / 资源 / 武将成长）
- 提供长期养成（城市发展 + 武将培养）
- 提供策略深度（官员分配 / 资源调配 / 外交博弈）
- 所有操作需分配武将执行，体现"人才是核心资源"

### 核心规则

> **所有内政操作必须分配武将/官员执行，未分配则该项不予更新。**

## 二、系统总架构

```
GameState (单例)
 └── CityProgress (每城池实例)
      ├── 资源管理 (Gold/Food/Wood/Iron/Population)
      ├── Buildings (Dictionary<string, Building>)
      ├── 官员分配 (Governor/InteriorOfficer/MilitaryOfficer/SearchOfficer)
      └── 驻扎武将列表

TurnManager (回合管理)
 ├── 每10日回合结算
 ├── 每月经济迭代（仅对已分配内政官的城池）
 └── 每季兵力迭代（仅对已分配军事官的城池）

InteriorConfig (静态配置表)
 ├── GetBuildingConfig(buildingId)
 ├── CalculateUpgradeCost(baseCost, targetLevel)
 └── CalculateProduction(config, level, cityScale, officerBonus)

DiplomacyManager (外交管理)
 └── 势力间关系（同盟/中立/敌对/通商）

CityActionDialog (UI层)
 ├── InteriorEconomy (经济概览)
 ├── InteriorDefense (防御信息)
 ├── InteriorBuilding (建筑升级)
 ├── OfficerAssignment (官员分配)
 ├── MilitaryManage (军事管理)
 ├── DiplomacyPanel (外交操作)
 └── EconomyManage (经济管理)
```

## 三、官员分配系统（核心机制）

### 3.1 职位体系

| 职位 | 关键属性 | 影响 | 每城数量 |
|------|----------|------|----------|
| 太守（城主） | 政治 + 统帅 | 城池总体效率 | 1 |
| 内政官 | 政治 | 资源产出加成（含商业收入） | 1 |
| 军事官 | 统帅 + 武力 | 征兵效率 + 防御 | 1 |
| 搜索官 | 智力 + 政治 | 发现武将概率 | 1 |

### 3.2 分配规则

- 每个武将同一时间只能担任一个城池的一个职位
- 已编入出征部队的武将不可分配为官员
- 俘虏状态的武将不可分配

### 3.3 官员效果

```
官员加成 = f(关键属性1, 关键属性2)
```

- **太守缺失**：城池所有内政效率降至50%
- **内政官缺失**：经济迭代时该城池不产出资源
- **军事官缺失**：兵力迭代时该城池不补充兵力
- **搜索官缺失**：无法执行搜索武将操作

### 3.4 官员特技加成

具有内政特技的武将担任官员时提供额外加成：
- **耕作**特技：粮食产出+15%
- **筑城**特技：建设速度+20%
- **商才**特技：金币收入+15%

## 五、建筑系统

### 建筑分类（5大类）

| 类别 | 建筑 | 效果 |
|------|------|------|
| 农业 | 农田、水利 | 提升粮食产出 |
| 商业 | 市场、商路 | 提升金币收入 |
| 军事 | 兵营、武库 | 提升征兵效率和装备品质 |
| 城防 | 城墙、箭塔 | 提升城池防御力 |
| 仓储 | 粮仓、金库 | 提升资源储存上限 |

> **注**：原4分类（Resource/Military/Functional/Tech）需重构为5大类。

### 建筑数据结构

```csharp
// BuildingConfig
public class BuildingConfig
{
    public string Id { get; set; }
    public string Name { get; set; }
    public BuildingCategory Category { get; set; }  // Agriculture/Commerce/Military/Defense/Storage
    public int MaxLevel { get; set; }
    public int BaseCost { get; set; }
    public ResourceType ProduceType { get; set; }
    public int BaseProduction { get; set; }
}
```

### 建筑升级

```
升级费用 = baseCost × 1.5^(targetLevel - 1)
产量 = baseProduction × level^1.2 × cityScaleMultiplier × officerBonus
```

- 升级建筑消耗即时扣除
- 官员加成：内政官(政治+经济)属性影响产量

### 建筑升级流程

```
玩家在CityActionDialog.InteriorBuilding阶段
 → 选中建筑
 → 再次点击触发升级
 → CityProgress.UpgradeBuilding(buildingId, out errorMsg)
   → 检查资源是否足够
   → 扣除资源
   → 建筑等级+1
```

## 六、生产系统（回合制）

### 核心

资源 = 回合函数（月度迭代）

> **重要变更**：原30秒连续Tick改为月度回合迭代。

### 产量计算模型

```
月度产量 = 基础产量 × 等级^1.2 × 城池规模加成 × 官员加成
```

- **官员加成**：`1.0 + 内政官政治 / 100`（无内政官则不产出）
- **太守加成**：太守存在时全效率，缺失时50%

### 城池规模加成（CityScaleConfig）

| 规模 | 加成倍率 | 最大兵力 | 守军槽位 |
|------|----------|----------|----------|
| small | 1.0x | 基础 | 1 |
| medium | 1.2x | +50% | 2 |
| large | 1.5x | +100% | 3 |
| huge | 2.0x | +200% | 4 |

### 生产执行

```
TurnManager检测月末
  → 遍历所有城池
  → 检查是否有内政官
  → 有：调用 CalculateMonthlyProduction() 增加资源
  → 无：跳过（不产出）
```

### 加成来源
- 建筑等级
- 城池规模（CityScaleConfig.GetProductionMultiplier）
- 内政官属性加成
- 武将特技加成（如耕作/商才）

## 七、军事管理（新增）

### 7.1 征兵系统

- **周期**：每3个月（兵力迭代）自动补充
- **前置条件**：城池已分配军事官
- **征兵量**：受兵营等级 × 人口基数 × 军事官(统帅+武力)加成
- **征兵上限**：受城池规模限制

### 7.2 训练系统

- 分配武将训练部队，提升兵种战斗力
- 训练效果受武将统帅属性影响
- 训练消耗粮食

### 7.3 装备管理

- 通过武库生产/分配装备
- 装备提升部队属性（通过Modifier系统）

## 八、外交系统（新增）

### 8.1 外交操作

| 操作 | 条件 | 效果 | 消耗 |
|------|------|------|------|
| 同盟 | 双方无敌对状态 | 共享视野、互不进攻 | 金币 |
| 通商 | 有同盟或中立 | 双方经济收入增加 | 无 |
| 停战 | 正在交战中 | 暂停敌对状态（有期限） | 金币/粮食 |
| 劝降 | 对方仅剩1~2城 | 尝试和平吞并弱势势力 | 大量金币 |

### 8.2 外交约束

- 外交操作需分配武将执行
- 成功率受武将政治和智力影响
- AI势力间也会进行外交操作
- DiplomacyManager管理势力间关系状态

### 8.3 外交关系

```
势力关系 = 敌对 / 中立 / 通商 / 同盟
```

- 关系影响AI攻击优先级
- 同盟关系有期限，到期可续约

## 九、经济管理（新增）

### 9.1 税率系统

| 税率 | 金币收入 | 民心影响 |
|------|----------|----------|
| 低税 | 基础×0.7 | 民心+5/月 |
| 标准 | 基础×1.0 | 无变化 |
| 重税 | 基础×1.5 | 民心-5/月 |

- 民心影响人口增长和叛乱概率

### 9.2 资源调配

- 城池间可进行资源转运
- 前置条件：两城间有路径联通
- 转运消耗时间（按行军距离）

### 9.3 人口管理

- 人口影响征兵上限和资源产出基数
- 人口自然增长受民心和粮食影响
- 战争导致人口减少

## 十、忠诚度与俸禄系统

### 10.1 忠诚度提升机制

忠诚度**主要通过势力君主经济赏赐提升**：

| 操作 | 忠诚度变化 | 说明 |
|------|------------|------|
| 赏赐金币 | +5~+15 | 按赏赐金额比例计算 |
| 赏赐装备 | +10~+20 | 按装备品质计算 |
| 升职（更高官位） | +10 | 获得更高官位时 |
| 战胜奖励 | +5 | 参与胜利战斗 |

### 10.2 忠诚度下降

| 事件 | 忠诚度变化 | 说明 |
|------|------------|------|
| 按时发放俸禄 | 维持不变 | 基本维系 |
| 长期不使用 | -2/月 | 连续3月未分配任务 |
| 俸禄不足 | -5/月 | 无法支付俸禄 |
| 被俘虏期间 | -3/月 | 关押状态 |
| 主公连续战败 | -5 | 连续3场战败 |

### 10.3 策反机制（重要）

忠诚度低于阈值时，敌对势力可策反劝走武将：

| 忠诚度范围 | 风险等级 | 策反成功率 |
|-----------|---------|-----------|
| >= 80 | 安全 | 不可策反 |
| 50~79 | 低风险 | 极低（5%以下） |
| 30~49 | 中风险 | (50 - 忠诚度) × 2% |
| < 30 | 高风险 | 显著提升，几乎必定被策反 |

- AI势力会主动对低忠诚度的敌方武将发起策反
- 势力君主魅力值影响麾下武将忠诚下降速率（高魅力君主减缓忠诚流失）
- 策反成功后武将携带装备转入敌方势力

### 10.4 俸禄系统

- 武将需要定期俸禄维持忠诚
- 俸禄金额与武将等级/官职挂钩
- 每月自动扣除俸禄，资金不足则忠诚度下降
- 高级武将俸禄更高

### 10.5 忠诚度影响

- 忠诚度 < 30：有叛变风险，可被敌方策反
- 忠诚度 < 50：被俘时容易投降，可被策反
- 忠诚度 > 80：被俘时难以招降，不可被策反

## 十一、人才系统

### 与内政系统整合

人才管理在 CityActionDialog 的 TalentManage 阶段：

| 操作 | 方法 | 费用 | 前置条件 |
|------|------|------|----------|
| 搜索 | `GameState.DiscoverTalent()` | 无（需搜索官） | 已分配搜索官，搜索官未行动 |
| 说服 | `GameState.PersuadeTalent()` | 无（按概率） | 目标为在野状态，已发现人才 |
| 招降 | `GameState.RecruitCaptive()` | 150战功 | 目标为俘虏状态 |

### 武将状态流转

```
历史登录 → Hidden（未出现）
    ↓（游戏日期到达 AppearYear 且在 AppearCity 范围）
Available（在野，隐藏）
    ↓（搜索官在对应城池发现）
Available（在野，可见）
    ↓（说服成功）
Recruited（已招募）→ 分配任务（战斗/内政/搜索）
    ↓（忠诚度过低 + 敌方策反）
转入敌方势力
    ↓（战斗失败）
俘获判定 → Captive（俘虏）
    ↓
招降 / 释放 / 关押
```

### 搜索机制（基于三国历史）

搜索系统严格遵循三国历史武将登场时间和地点：

#### 历史登场规则
- 每个武将在 generals.json 中配置 `AppearYear`（登场年份）和 `AppearCity`（登场城池）
- **时间约束**：游戏日期未到达 AppearYear 的武将不可被搜索到（处于 Hidden 状态）
- **地点约束**：武将只能在其 AppearCity 对应的城池或临近城池被发现
- 历史时间到达后，武将进入 Available 状态，等待被搜索发现

#### 搜索执行
- 搜索需要分配搜索官到城池
- 每回合手动触发搜索（消耗搜索官行动力）
- 搜索成功率受搜索官（智力+政治）影响
- 搜索结果只会出现该城池地理范围内、且已到达历史登场时间的在野武将

#### 搜索示例
```
赵云：AppearYear = 191, AppearCity = "changshan"
→ 191年前：Hidden状态，不可搜索
→ 191年后：在常山城池附近搜索可发现
→ 在洛阳搜索无法发现赵云（地点不匹配）
```

### 说服机制（羁绊优先）

说服成功率的计算**优先考虑武将间羁绊和势力关系**，魅力值仅作为辅助因素：

#### 说服成功率计算（优先级从高到低）

```
1. 羁绊关系判定（最高优先级）
   → 搜索官与目标武将存在历史羁绊（bonds.json）
   → 羁绊加成: +30%~+50%（根据羁绊类型）
   → 例: 关羽说服张飞 → 桃园结义羁绊 → +50%

2. 势力关系判定
   → 目标武将原属势力与当前势力关系
   → 同盟/友好势力: +10%
   → 敌对势力: -10%
   → 无势力（在野）: 无修正

3. 魅力值辅助（兜底因素）
   → 仅在无羁绊关系时作为主要参考
   → 有羁绊时魅力仅提供微小加成
   → 搜索官魅力 / 5（最高+20%）

4. 目标忠诚度修正
   → 目标忠诚度越低越容易说服
   → (100 - 目标忠诚度) / 200
```

#### 综合公式

```
说服成功率 = 基础概率(10%)
           + 羁绊加成(0~50%)     ← 核心因素
           + 势力关系(±10%)
           + 魅力辅助(0~20%)     ← 无羁绊时才显著
           + 忠诚度修正(0~50%)
最终 Clamp(5%, 95%)
```

#### 设计意图
- **有羁绊 → 高成功率**：鼓励玩家利用历史人物关系
- **无羁绊 → 依赖魅力**：作为通用兜底手段
- **势力关系 → 微调**：增加外交系统联动
- 说服成功后，被说服武将当前回合标记为"已行动"，下回合可用

## 十二、与战斗系统联动

### 内政 → 战斗

| 内政资源 | 战斗影响 |
|----------|----------|
| 资源 | 升级建筑、招募武将的基础 |
| 武将 | 编队出征的核心 |
| 城池规模 | 守军数量和防御加成 |
| 兵营等级 | 征兵效率 |
| 装备品质 | 部队属性加成 |

### 战斗 → 内政

| 战斗结果 | 内政影响 |
|----------|----------|
| 胜利占城 | 获得新城池及其资源 |
| 俘获武将 | 可招降加入，补充人才 |
| 战功积累 | 武将升级 → 选择1项属性加点（武/智/统/政/魅） |
| 失败 | 武将可能被俘，兵力损失 |

## 十五、数值模型

### 建筑升级费用
```
升级费用 = 基础费用 × 1.5^(目标等级 - 1)
```

| 等级 | 费用倍率 |
|------|----------|
| 1→2 | 1.0x |
| 2→3 | 1.5x |
| 3→4 | 2.25x |
| 5→6 | 5.06x |
| 9→10 | 38.4x |

### 月度产量计算
```
月产量 = 基础产量 × 等级^1.2 × 城池规模系数 × 官员加成
官员加成 = 1.0 + 内政官政治 / 100
```

### 设计原则
- 前期增长快，后期成本指数增长
- 资源消耗（建筑/招募/俸禄/征兵）与产出形成循环
- 城池规模提供乘法加成，鼓励多城发展
- 官员属性提供额外加成，体现人才价值

## 十六、最终总结

内政系统 = 回合制策略资源引擎
