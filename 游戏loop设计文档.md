# 猫三国 - 游戏Loop设计文档

> 最后更新：2026-04-15（根据需求文档同步）

## 一、设计目标

### 核心目标

构建一个回合制驱动的游戏主循环（Core Loop）：

- 以事件剧本为起点，以统一天下为终点
- 通过10日/月/季多层回合推进驱动游戏节奏
- 建立内政→战斗→扩张→成长闭环
- 所有内政操作需分配武将执行
- 支持数值扩展与版本更新

### 一句话定义

**玩家选择剧本和势力 → 通过回合制推进内政/外交/军事 → 占领所有城池统一天下**

## 二、核心Loop结构

### 主循环（Core Loop）

```
选择事件剧本 → 选择势力 → 进入大地图
    ↓
┌─────────────────────────────────────────┐
│ 玩家操作阶段                              │
│   分配官员 / 建设城池 / 外交 / 编队出征    │
│              ↓                           │
│   点击"结束回合" → 系统推进10日            │
│   ├─ 武将搜索/外交结算                    │
│   ├─ 行军推进 → 到达触发战斗              │
│   ├─ 历史武将登录检查                     │
│   ├─ （月末）经济迭代                     │
│   └─ （季末）兵力迭代                     │
│              ↓                           │
│   战斗 → 结算（奖励+俘获）→ 占领城池      │
│              ↓                           │
│   成长强化 → 继续下一回合                  │
└─────────────────────────────────────────┘
    ↓
一方势力占领所有城池 → 游戏结束（胜利/失败）
```

### 子循环

| 循环 | 周期 | 说明 |
|------|------|------|
| 短循环 | 单场战斗 | BattleScene（Deploy→Fight→Result+俘获） |
| 中循环 | 单回合(10日) | 操作→结束回合→系统结算 |
| 长循环 | 多回合(月/季/年) | 经济迭代→兵力迭代→扩张→统一 |

## 三、开局流程（新增）

### 3.1 剧本选择

```
主菜单 → "新建游戏"
       → 展示事件剧本列表（如"黄巾之乱 184年"/"官渡之战 200年"）
       → 选择一个剧本
```

每个剧本定义：
- 历史初始时间
- 各势力初始城池分布
- 各势力初始武将配置
- 各城池初始资源/兵力
- 已登录武将的初始归属

### 3.2 势力选择

```
选择剧本后 → 展示该剧本下所有可选势力
           → 显示势力信息（城池数/武将数/初始资源）
           → 选择势力 → 确认开始
```

### 3.3 初始化

```
ScenarioManager加载剧本数据
  → 初始化GameState（日期/城池/武将/资源/外交关系）
  → TurnManager初始化（当前日期 = 剧本起始日期）
  → 进入WorldMapScene
```

## 四、阶段拆解（回合制）

### 4.1 阶段一：玩家操作阶段（Player Phase）

玩家在回合内自由执行操作，无时间压力：

**内政操作**（通过CityActionDialog）

| 操作 | 前置条件 | 系统 |
|------|----------|------|
| 查看城池资源 | 无 | InteriorEconomy |
| 升级建筑 | 资源足够 | InteriorBuilding |
| 分配官员 | 有可用武将 | OfficerAssignment |
| 搜索武将 | 已分配搜索官 | TalentManage |
| 招募武将 | 目标为在野状态 | TalentManage |
| 管理俘虏 | 有俘虏 | CaptiveManage |

**军事操作**

| 操作 | 系统 |
|------|------|
| 编队出征 | MilitaryDeploy → SelectGeneral → SelectTarget → Confirm |
| 调兵换防 | ArmyManager |

**外交操作**

| 操作 | 条件 | 系统 |
|------|------|------|
| 同盟 | 双方无敌对 | DiplomacyManager |
| 通商 | 有同盟或中立 | DiplomacyManager |
| 停战 | 正在交战 | DiplomacyManager |
| 劝降 | 对方仅剩1~2城 | DiplomacyManager |

**结束回合**：点击"结束回合"按钮，进入系统结算阶段。

### 4.2 阶段二：系统结算阶段（System Phase）

点击"结束回合"后，TurnManager按以下顺序处理：

```
1. 推进游戏日期 +10日
2. 武将搜索结果结算（已分配搜索官的城池）
3. 外交行动结算
4. 行军推进（ArmyToken移动）
5. 到达目标 → 触发战斗（如有）
6. 历史武将登录检查（按当前日期）
7. 忠诚度变化结算（俸禄/俘虏/事件）
8. 如果到达月末 → 经济迭代
   - 仅对已分配内政官的城池结算资源产出
   - 税收/商业收入结算
   - 人口变化
9. 如果到达季末 → 兵力迭代
   - 仅对已分配军事官的城池结算征兵
   - 兵力恢复
   - 军备消耗
10. 胜利/失败条件检查
```

### 4.3 阶段三：战斗执行（Battle Phase）

当行军到达敌方城池时自动触发：

**流程**（BattleScene四阶段）
```
Deploy（查看部署）→ Countdown（3秒）→ Fighting（自动+手动技能）→ Result（结算+俘获）
```

**系统参与**

| 系统 | 职责 |
|------|------|
| BattleScene | Squad管理/伤害计算 |
| SkillTriggerSystem | 事件驱动触发链 |
| BuffSystem | 状态效果管理 |
| BattleAI | 敌方决策 |
| MoraleSystem | 士气变化 |
| AttributeSet + Modifier | 属性计算 |

**输出**
- 胜负结果 + 战斗数据
- 俘获判定结果

### 4.4 阶段四：奖励结算（Reward Phase）

**已实现的奖励计算**

| 奖励类型 | 计算公式 |
|----------|----------|
| XP | 基础200 + 杀敌×50 + 伤害比例 + 存活奖励 |
| 技能点 | 胜利2点/失败1点，3杀以上额外1点 |
| 技能XP | 使用次数×10 |
| 城池占领 | 胜利后占领目标城池（AddOwnedCity） |

**新增：俘获结算**

| 判定 | 说明 |
|------|------|
| 俘获概率 | 受武将速度/兵力比/忠诚度影响 |
| 未被俘获 | 武将返回己方最近城池 |
| 被俘获 | 进入俘虏(Captive)状态 |
| 后续处理 | 招降 / 释放 / 关押 |

**实现**：`BattleScene.AwardBattleRewards()` + `CaptureManager.ProcessCapture()`

### 4.5 阶段五：成长强化（Progression Phase）

**成长路径**

| 成长 | 触发条件 |
|------|----------|
| 武将经验(XP) | 参与战斗 |
| 武力提升 | 战斗/击杀 |
| 智力提升 | 使用技能/计策 |
| 统帅提升 | 率军/胜利 |
| 政治提升 | 执行内政任务 |
| 经济提升 | 管理城池商业 |
| 技能升级 | 积累技能XP |
| 建筑升级 | 消耗资源 |

## 五、系统交互关系（核心闭环）

```
内政系统 ──资源/武将──→ 战斗准备
    ↑                        ↓
    │                   战斗执行
    │                        ↓
    │                   俘获判定
    │                        ↓
    └───奖励/占城/俘虏── 奖励结算
```

| 方向 | 影响 |
|------|------|
| 内政→战斗 | 资源支撑武将培养、建筑影响兵种/征兵、官员属性影响效率 |
| 战斗→内政 | 占领新城池扩大资源来源、俘获武将可招降、XP提升属性 |
| 外交→战略 | 同盟减少敌人、通商增加收入、劝降加速统一 |
| 成长→战斗 | 更强武将→更高胜率→更难战役 |

## 六、数值循环设计

### 经济循环（回合制）

```
月度资源产出 ≥ 建筑升级消耗 + 武将俸禄 + 征兵消耗
```

- 资源"略紧张"，鼓励谨慎规划
- 多城发展提供规模优势

### 成长曲线

- 建筑升级费用：baseCost × 1.5^(level-1)（指数增长）
- 建筑产量：baseProduction × level^1.2（亚指数增长）
- 确保前期快速发展，后期需要多城池协同

## 七、时间节奏设计

| 节奏 | 时间跨度 | 对应操作 |
|------|----------|----------|
| 短期 | 1回合(10日) | 分配武将、出征、外交 |
| 中期 | 1月(3回合) | 经济迭代、建筑升级、搜索武将 |
| 长期 | 1季(9回合) | 兵力迭代、大规模战争、领土扩张 |
| 终局 | 1~N年 | 统一天下、收集名将 |

### 游戏阶段

| 阶段 | 时间 | 重点 |
|------|------|------|
| 前期（1年内） | ~36回合 | 稳固根据地，搜索武将，建设基础内政 |
| 中期（1~3年） | ~108回合 | 扩张领地，俘获/招降敌方武将，外交博弈 |
| 后期（3年+） | 不定 | 大规模战争，多线作战，统一天下 |

## 八、完整流程示例

```
1. 主菜单 → 新建游戏
2. 选择剧本"官渡之战 200年"
3. 选择势力"曹操" → 确认开始
4. 进入WorldMapScene（曹操初始城池/武将/资源已加载）

--- 第1回合 ---
5. 点击许昌 → 分配内政官（荀彧）→ 分配军事官（夏侯惇）
6. 点击许昌 → 建设 → 升级农田（消耗金币，粮食产量+）
7. 点击许昌 → 人才 → 搜索 → 发现郭嘉
8. 点击许昌 → 人才 → 招募郭嘉
9. 点击许昌 → 军事 → 编队（曹操+张辽+许褚）→ 选目标（陈留）→ 出征
10. 点击"结束回合"
11. 系统推进10日 → 军队向陈留行军
12. 历史武将登录检查（无新武将）

--- 第2回合 ---
13. 继续操作...点击"结束回合"
14. 军队到达陈留 → 触发战斗
15. Deploy→Countdown→Fighting→Result
16. 胜利 → 占领陈留 → 俘获敌将陈宫
17. 返回WorldMapScene
18. 管理俘虏 → 招降陈宫（消耗资源）

--- 第3回合（月末）---
19. 结束回合 → 经济迭代
20. 许昌（有内政官）产出资源
21. 陈留（未分配内政官）不产出
22. 分配陈宫为陈留内政官

... 持续推进直到统一天下
```

## 九、胜利/失败条件

| 条件 | 触发 |
|------|------|
| 胜利 | 玩家势力占领地图上全部城池 |
| 失败 | 玩家势力失去全部城池 |
| AI消灭 | AI势力间也会互相攻伐，势力可被消灭 |

每回合结算末尾检查胜利/失败条件。

## 十一、结论

游戏Loop = 回合制策略驱动引擎

核心循环从"30秒Tick连续产出"升级为"回合制多层迭代"（10日/月/季），更符合三国志11式的策略体验。玩家通过选择剧本和势力开局，在每回合中进行内政/外交/军事决策，通过占领城池、俘获武将、发展经济逐步统一天下。
