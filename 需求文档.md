# 猫三国 - 需求文档

> 最后更新：2026-04-15（根据最新需求叙述同步）

## 一、产品概述

### 1.1 产品定位
- **类型**：单机策略游戏（SLG + RTT）
- **玩法结构**：大地图战略 + 半实时战斗
- **参考作品**：三国志11（事件制 + 势力扮演 + 回合推进）
- **风格**：猫拟人 + 三国题材 + 轻策略深系统 + 2D平面

### 1.2 目标用户

| 用户类型 | 特征 |
|----------|------|
| 策略玩家 | 喜欢《三国志11》《三国群英传》《全面战争》 |
| 轻度玩家 | 可接受自动战斗 |
| 三国爱好者 | 对武将养成和历史事件有兴趣 |

### 1.3 核心卖点
- 猫拟人三国（差异化题材）
- 三国志11式事件开局系统（多剧本高复玩）
- 技能触发链战斗系统（深度玩法）
- 半自动战斗（低操作，高策略）
- 城池经营 + 扩张（内政需分配武将执行）
- 武将不死亡机制 + 俘获招降（策略深度）

### 1.4 核心玩法循环
```
选择事件剧本 → 选择势力 → 开始游戏
    ↓
┌─────────────────────────────────────────┐
│  每10日推进：搜索武将 / 分配官员 / 外交  │
│  每1月迭代：城池经济产出                  │
│  每3月迭代：兵力恢复与征兵                │
│  随时可触发：出征 → 战斗 → 占领          │
└─────────────────────────────────────────┘
    ↓
一方势力占领所有城池 → 游戏结束（胜利/失败）
```

### 1.5 胜利条件
- **胜利**：玩家势力占领地图上所有城池
- **失败**：玩家势力失去所有城池
- **其他势力**：AI势力同样遵循此规则，互相攻伐

## 二、技术架构（MonoGame 2D）

- **游戏框架**：MonoGame (XNA) - C# 跨平台2D游戏框架
- **渲染方式**：SpriteBatch 2D渲染，预乘Alpha混合
- **字体系统**：FontStashSharp（SimHei 14/20/32/42pt）
- **数据格式**：JSON配置驱动（System.Text.Json）
- **目标分辨率**：1280×720，支持窗口动态缩放

## 三、时间与事件系统（核心机制）

### 3.1 事件剧本系统（参考三国志11）

游戏开局流程：
```
主菜单 → 选择事件剧本 → 选择势力 → 开始游戏
```

**事件剧本**是游戏的起始时间点，决定：
- 历史初始时间（如"黄巾之乱 184年"、"官渡之战 200年"）
- 各势力初始城池分布
- 各势力初始武将配置
- 各城池初始资源/兵力
- 已登录武将的初始归属

剧本数据从 JSON 配置加载，每个剧本包含完整的世界初始状态快照。

### 3.2 时间推进机制

游戏采用**回合推进制**，以"日"为最小时间单位：

| 周期 | 触发内容 |
|------|----------|
| **每10日** | 一次回合更新：武将行动、搜索、外交、事件触发 |
| **每1月** | 城池经济迭代：资源产出、建筑收益、人口变化 |
| **每3月** | 兵力迭代：征兵恢复、兵力补充、军备消耗 |

- 玩家可在回合内执行操作（分配武将、调兵、出征等）
- 点击"结束回合"推进10日
- 经济和兵力迭代在对应月/季节点自动触发

### 3.3 武将历史登录

武将根据历史时间点在相关城池自动登录（进入在野状态）：
- 每个武将配置有"出现年份"和"出现城池"
- 当游戏时间推进到该年份时，武将在对应城池变为可发现状态
- 通过**搜索**操作（需分配武将执行）可发现在野武将
- 未被任何势力发现的武将保持隐藏

## 四、大地图系统（策略核心）

### 4.1 城池节点（Node-based）

- 非自由移动，点对点行军
- 16×10虚拟网格，映射到2000×1400世界坐标
- 城池数据从 `cities.json` 加载
- 城池属性：位置、归属势力、规模(small/medium/large/huge)、防御、资源

### 4.2 地图交互（已实现）

- **相机系统**：Camera2D支持滚轮缩放(0.5x~1.5x)、自适应满屏缩放
- **拖拽平移**：左键拖拽移动地图（5px阈值区分点击和拖拽）
- **城池选择**：左键点击城池打开CityActionDialog
- **HUD偏移**：顶部60px HUD区域补偿，确保上方城池可选

### 4.3 渲染层（已实现）

| 渲染器 | 职责 |
|--------|------|
| `MapBackgroundRenderer` | 地图背景纹理 |
| `ProvinceRenderer` | 州郡区域 |
| `TerrainRenderer` | 地形特征（山脉/河流/资源点） |
| `RoadRenderer` | 道路绘制（陆路3px土黄/水路2px蓝色虚线） |
| `CityRenderer` | 城池节点绘制 |

### 4.4 功能模块

**1. 城池内政系统**

> **核心规则：所有内政操作必须分配武将/官员执行，未分配则该项不予更新。**

- 资源：金、粮、木、铁、人口
- 建筑：升级系统（费用指数增长 baseCost × 1.5^(level-1)）
- 产量：baseProduction × level^1.2 × cityScaleMultiplier
- 经济产出周期：**每1个月**自动结算一次（仅对已分配官员的城池生效）
- 兵力恢复周期：**每3个月**自动结算一次（仅对已分配武将的城池生效）

建筑分类：
| 类别 | 建筑 | 效果 |
|------|------|------|
| 农业 | 农田、水利 | 提升粮食产出 |
| 商业 | 市场、商路 | 提升金币收入 |
| 军事 | 兵营、武库 | 提升征兵效率和装备品质 |
| 城防 | 城墙、箭塔 | 提升城池防御力 |
| 仓储 | 粮仓、金库 | 提升资源储存上限 |

官员分配规则：
| 职位 | 影响 | 关键属性 |
|------|------|----------|
| 太守（城主） | 城池总体效率 | 政治 + 统帅 |
| 内政官 | 资源产出加成 | 政治 + 经济 |
| 军事官 | 征兵效率 + 防御 | 统帅 + 武力 |
| 搜索官 | 发现武将概率 | 智力 + 政治 |

- 未分配太守的城池：所有内政效率降至50%
- 未分配内政官的城池：经济迭代时不产出资源
- 未分配军事官的城池：兵力迭代时不补充兵力

**2. 军事管理**
- 征兵：每3个月兵力迭代时补充，征兵量受兵营等级和人口影响
- 训练：分配武将训练部队，提升兵种战斗力（受武将统帅影响）
- 装备管理：通过武库生产/分配装备，提升部队属性

**3. 外交系统**

| 外交操作 | 条件 | 效果 |
|----------|------|------|
| 同盟 | 双方无敌对状态 | 共享视野、互不进攻 |
| 通商 | 双方有同盟或中立 | 双方经济收入增加 |
| 停战 | 正在交战中 | 暂停敌对状态（有期限） |
| 劝降 | 对方仅剩1~2城 | 尝试和平吞并弱势势力 |

- 外交操作需分配武将执行，受武将政治和智力影响
- AI势力间也会进行外交操作

**4. 经济管理**
- 税率调整：提高税率增加金币但降低民心，降低税率反之
- 资源调配：城池间可进行资源转运（需行军路线联通）
- 人口管理：人口影响征兵上限和资源产出基数

**5. 行军系统**
- A→B点对点寻路（MapPathfinder）
- ArmyManager管理军队Token移动
- 到达敌方城池触发自动战斗

**6. 雾战系统**
- FogOfWarManager根据玩家城市和军队更新可见区域
- 不可见城池和道路不绘制

## 五、战斗系统（重点设计）

### 半实时战斗（RTT）

采用"Squad（方阵）"模型，非逐单位模拟。

### 战斗流程（已实现）

```
Deploy（部署）→ Countdown（3秒倒计时）→ Fighting（战斗）→ Result（结算）
```

### 战斗组成

**1. 阵型系统（9种，已实现）**
- 先锋阵、鱼鳞阵、方阵、锥形阵、长蛇阵
- 鹤翼阵、八卦阵、偃月阵、环形阵
- 每种阵型有独特特殊能力（伤害分摊/穿透/包围加成等）

**2. 军种系统（10种，已实现）**
- 步兵、枪兵、盾兵、骑兵、重骑、轻骑
- 弓兵、强弩、攻城、法师
- 10×10克制表，系数范围0.5~1.6倍

**3. 战斗核心三件套（已实现）**
- **EventBus**：事件发布/订阅（OnBattleStart等）
- **BuffSystem**：BUFF状态管理和持续时间
- **SkillTriggerSystem**：技能触发链系统

**4. 属性系统（已实现）**
- AttributeSet + Modifier系统
- 支持加法和乘法修饰器
- 最终值 = (基础值 + 固定加成) × 百分比

**5. 战斗算法**
```
伤害 = 攻击力 × 技能系数 × 克制系数 × 随机浮动
```

**6. 士气系统（已实现）**
- MoraleSystem：武将阵亡降低全军士气
- 影响战斗表现和逃跑判定

**7. 战斗AI（已实现）**
- BattleAI：支持难度等级
- 选目标 + 技能释放判断

### 奖励系统（已实现）
- XP：基础200 + 杀敌×50 + 伤害比例 + 存活奖励
- 技能点：胜利2点/失败1点，3杀以上额外1点
- 技能XP：使用次数×10

### 战斗结算与武将俘获

> **核心规则：武将不会在战斗中死亡。**

战斗结束时对失败方武将进行俘获判定：
- 失败方每个武将有一定概率被胜利方**俘获**
- 俘获概率受以下因素影响：
  - 武将速度（越高越难被抓）
  - 战斗剩余兵力比（惨败更容易被俘）
  - 武将忠诚度（低忠诚更容易投降）
- 未被俘获的武将返回己方最近城池
- 被俘武将进入**俘虏(Captive)**状态，胜利方可选择：
  - **招降**：消耗资源尝试招募（受忠诚度影响）
  - **释放**：增加外交好感
  - **关押**：持续关押，忠诚度逐步降低

## 七、武将系统（核心卖点）

### 7.1 属性设计（6维）

| 属性 | 代码字段 | 作用 | 影响范围 |
|------|---------|------|----------|
| 武力 | Strength | 影响物理伤害 | 战斗攻击力 |
| 智力 | Intelligence | 影响技能效果 | 计策成功率、技能伤害 |
| 统帅 | Command | 影响带兵能力 | 士气、部队规模上限 |
| 政治 | Politics | 影响内政效率 | 资源产出、建设速度 |
| 经济 | Economics | 影响经济收入 | 城池商业/税收加成 |
| 忠诚度 | Loyalty | 影响忠诚稳定 | 被俘投降概率、叛变概率 |

- 武力/智力/统帅/政治/经济 为培养型属性，范围 1~100
- 忠诚度为动态属性，范围 0~100，受事件/俘虏/待遇影响浮动
- **武将能力数值在后续统一配置**（JSON数据驱动）

### 7.2 武将特技系统

部分武将初始时具备特技（特殊被动能力）：

| 特技类型 | 说明 | 示例 |
|----------|------|------|
| 战斗特技 | 战斗中自动触发 | 神射（远程伤害+20%）、突击（首次冲锋伤害+50%） |
| 内政特技 | 内政时提供加成 | 耕作（粮食产出+15%）、筑城（建设速度+20%） |
| 指挥特技 | 影响部队整体 | 鼓舞（士气回复加速）、铁壁（全军防御+10%） |

- 特技在武将数据中配置，初始自带
- **将领能力/数值/特技在后续统一配置**

### 7.3 武将生命周期

```
历史登录 → 在野(Hidden) → 被搜索发现(Available)
    ↓
说服(PersuadeTalent) → 已招募(Recruited) → 分配任务（战斗/内政/搜索）
    ↓                                         ↓
                                    战斗失败 → 俘获判定
                                               ↓
                                    俘虏(Captive) → 招降 / 释放 / 关押
```

### 7.4 武将培养

武将通过实际执行任务获得经验，多维度独立成长：
- **武力提升**：参与战斗、取得击杀
- **智力提升**：使用技能、执行计策
- **统帅提升**：率领大部队、取得战斗胜利
- **政治提升**：执行内政任务
- **经济提升**：管理城池商业
- **忠诚度管理**：
  - 提升：赏赐金币/装备、升职、战胜后奖励
  - 降低：长期不使用、俸禄不足、被俘虏期间
  - 俸禄系统：武将需要定期俸禄维持忠诚，高级武将俸禄更高

### 7.5 技能系统
- 主动技能（大招）+ 被动技能（Buff）

## 十三、游戏流程设计

### 13.1 开局流程
```
主菜单 → 选择事件剧本（如"黄巾之乱 184年"/"官渡之战 200年"等）
       → 选择势力（查看各势力初始城池/武将/资源）
       → 确认开始
       → 进入大地图（世界按剧本初始状态加载）
```

### 13.2 回合循环
```
┌───────────────────────────────────┐
│         玩家操作阶段               │
│  ├─ 查看城池状态                  │
│  ├─ 分配武将（内政/军事/搜索）     │
│  ├─ 编队出征                      │
│  ├─ 外交操作                      │
│  └─ 查看/管理俘虏                 │
│         ↓                         │
│    点击"结束回合"                  │
│         ↓                         │
│    系统推进10日                    │
│  ├─ 武将搜索结果结算              │
│  ├─ 行军推进                      │
│  ├─ 到达目标触发战斗              │
│  ├─ 历史武将登录检查              │
│  ├─ （月末）经济迭代              │
│  └─ （季末）兵力迭代              │
└───────────────────────────────────┘
```

### 13.3 胜利/失败条件
- **胜利**：玩家势力占领地图上全部城池
- **失败**：玩家势力失去全部城池
- AI势力间也会互相攻伐，可能有势力被消灭

### 13.4 长期目标
- 统一全部城池
- 收集历史名将
- 构建最强阵容搭配
- 完成不同剧本挑战

## 十四、节奏设计

### 战斗节奏

| 阶段 | 时长 |
|------|------|
| 部署 | 5秒 |
| 战斗 | 30~90秒 |

### 回合节奏

| 阶段 | 说明 |
|------|------|
| 前期（1年内） | 稳固根据地，搜索武将，建设基础内政 |
| 中期（1~3年） | 扩张领地，俘获/招降敌方武将 |
| 后期（3年+） | 大规模战争，多线作战，统一天下 |

### 资源节奏
- 经济每月结算，资源始终"略紧张"
- 兵力每季恢复，鼓励谨慎用兵而非人海战术
- 武将培养需要持续分配任务，不可速成
