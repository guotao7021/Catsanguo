十、战斗结算模型（核心）

> **实现状态**: ✅ 已实现于 `DamageCalculator.Calculate()` + `Squad.TakeDamage()`。

10.1 实际伤害公式（DamageCalculator 第11-58行）
```
damage = baseAttack × skillCoefficient
       × unitCounterMod          // 军种克制（查10×10矩阵）
       × formationMod            // 阵型克制（±10%，骑兵>弓兵>步兵>骑兵）
       × formationAtkBonus       // 阵型攻击加成（如锥形阵+30%）
       × unitAtkBonus            // 军种攻击倍率（如重骑×1.5）
       / (formationDefBonus × unitDefBonus)  // 防御方加成（除法）
       × randomVariance          // 随机波动 0.9~1.1
       × moraleMod               // 士气修正 0.5~1.0
```

特殊修正（按顺序追加）：
1. 远程 vs 前排（Archer → Vanguard）：×0.6（40%减免）
2. 阵型减伤（如鱼鳞阵DamageReduction=0.4）：×(1 - DamageReduction)
3. 骑兵首次冲锋：×1.5（`Squad.PerformAttack()` 中计算）

10.2 受伤公式（Squad.TakeDamage 第410-421行）
```
实际伤害 = damage × (100 / (100 + EffectiveDefense))
```
此公式为软减伤模型：防御100时减伤50%，防御200时减伤66.7%，防御300时减伤75%。

```
俘获概率 = 基础概率(20%) + (追击方智力 - 逃跑方智力) × 1%
```
- 最终 Clamp(10%, 60%)

2. 建筑回合制产出（`ProduceBuildingResources`）
3. 征兵处理（`ProcessRecruitment`）
4. 远程策反任务处理
5. 月末事件（`TriggerMonthEnd`）：俸禄扣除 + 官员加成
6. 季末事件（`TriggerQuarterEnd`）：季度兵力恢复

12.2 粮草消耗（ConsumeGarrisonGrain 第197-215行）
```
每回合粮草消耗 = Max(1, CurrentTroops / 10)
```
粮尽（Grain < 0）时逃兵：`desertCount = Min(CurrentTroops, |Grain| × 2)`

12.3 建筑产出（ProduceBuildingResources 第220-249行）
- 通过 `CityProgress.GetTotalProduction(cityScale)` 计算
- 产出公式：`InteriorConfig.CalculateProduction(config, level, cityScale)`
  = `config.BaseProduction × level^1.2 × cityScaleMultiplier`
- 兵营特殊处理：每回合产兵 = `barracks.Level × 3`

12.4 征兵系统（ProcessRecruitment 第255-315行）
```
每回合征兵量 = 10 + barracksLevel × 5 + 军事官统帅/10
征兵消耗 = 征兵量 × 2 金币 + 征兵量 × 1 粮草
```
- 资源不足时部分征兵
- 达到目标兵力后停止征兵

12.5 月末官员加成（TriggerMonthEnd）

**内政官加成**（第117-127行）：
```
bonus = 内政官政治值 / 10
城池 Gold += bonus, Food += bonus
```

**军事官加成**（第130-143行）：
```
troopBonus = 军事官统帅 / 5
CurrentTroops = Min(CurrentTroops + troopBonus, MaxTroops)
```

12.6 季末兵力恢复（TriggerQuarterEnd 第150-183行）
```
baseRecovery = 20 + 太守统帅 / 10
CurrentTroops = Min(CurrentTroops + baseRecovery, MaxTroops)
```

十八、一句话总结
该数值体系通过"兵力×单位质量×组合修正"实现可预测且具策略深度的SLG战斗模型。核心伤害公式采用多层乘法（军种克制 × 阵型修正 × 军种倍率 × 士气），防御采用软减伤模型 `damage × 100/(100+DEF)`。武将通过战功升级（每级+3%属性），忠诚度系统部分实现（俸禄不足扣忠诚），俘虏系统完整（撤退/俘获/招降/释放），外交系统已上线（敌对/中立/贸易/同盟/停战）。
