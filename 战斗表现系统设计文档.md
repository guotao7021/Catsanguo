一、系统定位与目标
🎯 核心定位

战斗表现系统 = 把“计算结果”转化为“可视化体验”

🎯 必须实现
✅ 逻辑与表现彻底解耦（工业级标准）
✅ 支持技能、Buff、阵形的视觉表达
✅ 支持回放系统（同步表现）
✅ 支持高并发（100+单位）
🧱 二、系统总体架构
CombatPresentationSystem
 ├── EventReceiver（接收战斗事件）
 ├── AnimationSystem（动画）
 ├── VFXSystem（特效）
 ├── HitSystem（打击反馈）
 ├── UIEffectSystem（数值飘字/UI）
 ├── CameraSystem（镜头）
 ├── AudioSystem（音效）
🧠 三、核心设计思想（工业级关键）
❗1. 表现 ≠ 逻辑
Simulation（计算） → Event → Presentation（表现）
❗2. 一切表现由“事件驱动”
🎯 战斗事件（核心）
public enum CombatEventType
{
    Attack,
    Hit,
    Crit,
    SkillCast,
    BuffAdd,
    BuffRemove,
    UnitDeath
}
📡 四、事件驱动系统（核心）
4.1 Event结构
public struct CombatEvent
{
    public CombatEventType type;
    public int sourceId;
    public int targetId;
    public float value; // 伤害/治疗
}
4.2 数据流
Simulation.Tick()
   ↓
产生 CombatEvent
   ↓
PresentationSystem消费
   ↓
触发动画/特效/UI
🎭 五、动画系统（AnimationSystem）
🎯 角色状态机
Idle → Move → Attack → Skill → Hit → Dead
🎯 动画触发
if (event.type == Attack)
    PlayAnimation("Attack");
🎯 动画层级（推荐）
Base Layer（基础动作）
Upper Body（技能动作）
Additive Layer（受击反馈）
✨ 六、特效系统（VFXSystem）
🎯 特效分类
类型	示例
攻击	刀光
技能	火球
Buff	光环
环境	爆炸
🎯 绑定方式
特效 = SkillID / BuffID 绑定
🎯 示例
PlayVFX("Fireball", source, target);
💥 七、打击反馈系统（HitSystem）
🎯 核心体验来源（非常重要）
7.1 命中反馈
停顿（Hit Stop）
震屏（Camera Shake）
闪白（Flash）
7.2 示例实现
if (event.type == Hit)
{
    Camera.Shake(0.2f);
    Time.timeScale = 0.9f;
}
🎯 Crit强化
暴击 → 更强震动 + 更大字体
🔢 八、UI表现系统（UIEffectSystem）
🎯 飘字
-100（白色）
-300（红色暴击）
+50（绿色治疗）
🎯 示例
ShowDamageText(value, position);
🎯 Buff提示
图标
倒计时
层数
🎥 九、镜头系统（CameraSystem）
