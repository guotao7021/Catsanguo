# 猫三国 - 战斗表现系统设计文档

> 最后更新：2026-04-15（根据需求文档同步）

## 一、系统定位与目标

### 核心定位

战斗表现系统 = 把"计算结果"转化为"可视化体验"

### 已实现能力
- 逻辑与表现通过EventBus解耦
- 支持技能、Buff视觉表达
- 支持浮动伤害数字
- 支持屏幕震动反馈
- 支持阵亡通知

## 二、系统总体架构（已实现）

```
BattleScene
 ├── EventBus（事件接收）
 │    └── 战斗事件 → 触发表现
 ├── Squad渲染（精灵+血条+名称）
 ├── FloatingTextManager（飘字系统）
 ├── SkillVFX（技能特效）
 ├── DeathNotificationManager（阵亡通知）
 └── ScreenShake（屏幕震动）
```

## 三、核心设计思想

### 1. 表现 ≠ 逻辑

```
BattleContext (计算) → EventBus → BattleScene (表现)
```

战斗计算（伤害/Buff/状态）在逻辑层完成，通过EventBus发布事件，表现层订阅事件并播放对应视觉效果。

### 2. 事件驱动

所有表现由战斗事件触发：

| 事件 | 表现 |
|------|------|
| OnAttack | 播放攻击动画 |
| OnHit | 显示伤害飘字 + 震屏 |
| OnCrit | 更大字体 + 更强震动 |
| OnSkillCast | SkillVFX环形特效 |
| OnBuffAdd | 状态图标显示 |
| OnRetreat | 撤退通知 + 强烈震屏（武将不死亡） |
| OnCapture | 俘获提示（新增） |
| OnBattleStart | 中等震屏 |

## 四、已实现的表现系统

### 4.1 Squad精灵渲染

```csharp
private void DrawSquad(Squad squad, Vector2 offset)
```

- 在世界坐标位置绘制武将精灵
- 血条（HP百分比）
- 武将名称文字

### 4.2 飘字系统（FloatingTextManager）

| 类型 | 颜色 | 效果 |
|------|------|------|
| 普通伤害 | 白色 | 向上漂浮+渐隐 |
| 暴击伤害 | 红色/大字体 | 向上漂浮+渐隐 |
| 治疗 | 绿色 | 向上漂浮+渐隐 |

**实现机制**：
- 创建文字实例于伤害/治疗发生位置
- 每帧向上移动 + 透明度递减
- 超过生命周期自动移除

### 4.3 技能VFX（SkillVFX）

```csharp
private void AddSkillVFX(Squad caster)
```

- 技能释放时在施法者位置创建视觉效果
- 环形扩展效果（半径随时间增大）
- 渐隐消失

### 4.4 屏幕震动

在BattleScene中实现的震屏效果：

| 触发 | 强度 | 持续时间 |
|------|------|----------|
| 战斗开始 | 中等 | 0.3s |
| 技能释放 | 较强 | 0.2s |
| 武将撤退 | 强烈 | 0.4s |

**实现方式**：相机偏移 + 随机方向 + 衰减

### 4.5 撤退通知（RetreatNotificationManager）

> **重要变更**：武将不会在战斗中死亡，原"阵亡通知"改为"撤退通知"。

- 武将HP归零时在屏幕上方弹出撤退通知
- 显示武将名称和撤退信息
- 自动倒计时消失

### 4.6 俘获表现（新增）

战斗结束后的俘获判定表现：

| 表现 | 说明 |
|------|------|
| 俘获成功提示 | 飘字"俘获 [武将名]"（金色） |
| 逃脱提示 | 飘字"[武将名] 逃脱"（灰色） |
| 俘获面板 | 结算时显示所有俘获结果和处理选项 |

## 五、渲染管线

```
BattleScene.Draw()
 ├── SpriteBatch.Begin(transformMatrix)  // 世界空间
 │    ├── 战场背景
 │    ├── DrawSquad() × N  // 所有方阵
 │    └── SkillVFX绘制
 ├── SpriteBatch.End()
 └── SpriteBatch.Begin()  // 屏幕空间
      ├── HUD
      ├── FloatingText绘制
      ├── DeathNotification绘制
      └── ResultPanel
```

### 渲染技术

| 技术 | 说明 |
|------|------|
| SpriteBatch | MonoGame 2D批量渲染 |
| 预乘Alpha | Color * float 实现透明度 |
| 矩阵变换 | Camera2D提供世界→屏幕变换 |

## 六、与战斗系统联动

```
DamageCalculator → 计算伤害
    ↓
EventBus.Publish(OnHit, damage)
    ↓
FloatingTextManager.Add(value, position, color)
    ↓
ScreenShake(intensity)
```

```
SkillTriggerSystem → 触发技能
    ↓
EventBus.Publish(OnSkillCast)
    ↓
AddSkillVFX(caster)
    ↓
ScreenShake(intensity)
```

```
MoraleSystem → 武将撤退
    ↓
EventBus.Publish(OnRetreat)
    ↓
RetreatNotificationManager.Add(generalName)
    ↓
ScreenShake(strong)
```

```
CaptureManager → 俘获判定（战斗结束时）
    ↓
EventBus.Publish(OnCapture, capturedGenerals)
    ↓
CaptureResultPanel显示俘获结果
    ↓
玩家选择：招降/释放/关押
```

## 七、性能特点

| 方面 | 策略 |
|------|------|
| 渲染开销 | 2D精灵渲染，极低GPU开销 |
| 飘字管理 | 有限生命周期，自动回收 |
| VFX管理 | 简单几何图形，无纹理需求 |
| 批量绘制 | SpriteBatch自动合批 |

## 九、最终总结

战斗表现系统 = 战斗体验的80%来源

当前已通过EventBus实现逻辑与表现的解耦，提供飘字、技能VFX、屏幕震动、撤退通知等基础视觉反馈。术语从"阵亡"改为"撤退"以匹配武将不死亡机制，新增俘获判定表现和CaptureResultPanel。所有渲染基于MonoGame SpriteBatch 2D渲染，性能开销极低。
