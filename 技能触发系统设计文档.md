一、技能触发链系统目标
✅ 要解决的问题
技能不只是“点一下放一下”
支持：
连锁触发（A触发B）
条件触发（暴击才触发）
被动技能（全程生效）
复杂组合（类似卡牌/ARPG）
🎯 一句话定义

技能 = 触发器（Trigger） + 条件（Condition） + 效果（Effect）链

🧱 二、系统总架构
SkillSystem
 ├── SkillInstance
 ├── TriggerSystem
 ├── ConditionSystem
 ├── EffectExecutor
 ├── EventBus（复用）
数据流（核心）
事件(Event)
   ↓
Trigger匹配
   ↓
Condition判断
   ↓
Effect执行
   ↓
（可能再次触发Event → 链式反应）
🧠 三、核心设计思想（工业级关键）
❗1. 技能完全数据驱动
不写技能代码 → 全部配置
❗2. 触发链 = 事件驱动图（Event Graph）
❗3. 技能不直接做事

👉 技能只负责：

监听事件 → 判断 → 执行Effect

🧩 五、Trigger系统设计
🎯 作用

监听事件 → 找到匹配技能

支持事件类型
public enum EventType
{
    OnAttack,
    OnHit,
    OnKill,
    OnDamaged,
    OnBuffAdd,
    OnSkillCast
}
Trigger结构
public struct SkillTrigger
{
    public EventType eventType;
    public Condition[] conditions;
    public Effect[] effects;
}
⚖️ 六、Condition系统（核心扩展点）
🎯 常见Condition
类型	示例
概率	30%触发
血量	HP < 50%
距离	距离<5
Buff存在	有中毒
暴击	暴击时
接口设计
public interface ICondition
{
    bool Check(EventContext ctx);
}
示例：概率触发
public class RandomCondition : ICondition
{
    public float probability;

    public bool Check(EventContext ctx)
    {
        return ctx.Random() < probability;
    }
}
⚔️ 七、Effect执行系统
🎯 Effect类型
类型	示例
AddBuff	加Buff
Damage	伤害
Heal	治疗
Spawn	召唤
ModifyAttr	改属性
接口
public interface IEffect
{
    void Execute(EventContext ctx);
}
示例：添加Buff
public class AddBuffEffect : IEffect
{
    public int buffId;

    public void Execute(EventContext ctx)
    {
        ctx.world.AddBuff(buffId, ctx.sourceId, ctx.targetId);
    }
}
🔄 八、触发链执行流程（核心）
完整链路
单位攻击
 → 触发 OnAttack
 → SkillSystem扫描技能
 → 匹配Trigger
 → 检查Condition
 → 执行Effect
 → Effect产生新Event
 → 继续触发（链式）
示例链：
攻击
 → 30%概率触发中毒
 → 中毒Buff OnTick
 → 造成伤害
 → 触发 OnDamaged
 → 被动技能触发回血
⚠️ 九、循环触发保护（必须做）
❗问题

无限触发：

A → B → A → B → 死循环
解决方案
1️⃣ 深度限制
if (ctx.depth > MAX_DEPTH) return;
2️⃣ 触发标记
同一帧同一技能只触发一次
⚡ 十、性能优化（工业级）
1️⃣ Trigger索引表（关键）
EventType → Skill列表

👉 不要每次遍历全部技能

2️⃣ 条件短路
一个Condition失败 → 直接退出
3️⃣ Skill缓存
单位持有技能列表缓存
