# 猫三国 - 技能触发系统设计文档

> 最后更新：2026-04-15（根据实际代码同步）

## 一、技能触发链系统目标

### 要解决的问题
技能不只是"点一下放一下"，支持：
- 连锁触发（A触发B）
- 条件触发（暴击才触发）
- 被动技能（全程生效）
- 复杂组合

### 一句话定义

**技能 = 触发器（Trigger） + 条件（Condition） + 效果（Effect）链**

## 二、系统总架构（已实现）

```
BattleScene
 ├── EventBus（事件发布/订阅）
 ├── SkillTriggerSystem（技能触发链执行）
 ├── BuffSystem（Buff状态管理）
 └── BattleContext（连接所有子系统）
```

### 数据流

```
战斗事件(Event)
   ↓
SkillTriggerSystem.OnEvent()
   ↓
匹配Trigger
   ↓
检查Condition
   ↓
执行Effect
   ↓
（可能产生新Event → 链式反应）
```

## 三、核心设计思想

### 1. 技能完全数据驱动

技能逻辑通过JSON配置定义，不需要为每个技能写代码。

### 2. 触发链 = 事件驱动图

所有技能行为都是对战斗事件的响应。

### 3. 技能只负责

**监听事件 → 判断条件 → 执行效果**

## 五、事件类型（已实现）

通过EventBus发布的战斗事件：

| 事件 | 触发时机 |
|------|----------|
| OnBattleStart | 战斗开始时 |
| OnAttack | 发起攻击时 |
| OnHit | 命中目标时 |
| OnCrit | 暴击时 |
| OnKill | 击杀时 |
| OnDamaged | 受到伤害时 |
| OnBuffAdd | 获得Buff时 |
| OnBuffRemove | Buff移除时 |
| OnSkillCast | 释放技能时 |
| OnDeath | 死亡时 |

## 六、Condition系统

### 常见条件类型

| 类型 | 说明 | 示例 |
|------|------|------|
| Random | 概率触发 | 30%触发 |
| HpBelow | 血量低于阈值 | HP < 50% |
| HpAbove | 血量高于阈值 | HP > 80% |
| BuffExists | 目标有特定Buff | 有中毒状态 |
| IsCrit | 是否暴击 | 暴击时触发 |
| CooldownReady | CD已冷却 | 技能可用 |

### 条件检查

条件支持短路计算：一个Condition失败→直接退出，不执行Effect。

## 八、触发链执行流程

### 完整链路

```
单位攻击
 → EventBus发布 OnAttack
 → SkillTriggerSystem 扫描该单位技能
 → 匹配Trigger（event == OnAttack）
 → 检查Condition（Random(0.3)）
 → 执行Effect（AddBuff(中毒)）
 → BuffSystem添加中毒Buff
 → 中毒每Tick造成伤害
 → EventBus发布 OnDamaged
 → 触发被动技能（如：受伤回血）
```

### 典型技能示例

**普攻附带中毒**
```json
{ "event": "OnHit", "conditions": [{"type": "Random", "value": 0.3}], "effects": [{"type": "AddBuff", "buffId": "poison"}] }
```

**残血回血**
```json
{ "event": "OnDamaged", "conditions": [{"type": "HpBelow", "value": 0.3}], "effects": [{"type": "Heal", "value": 50}] }
```

**击杀连锁**
```
OnKill → AOE Damage → OnDamaged → 连锁触发
```

## 九、循环触发保护

### 防止无限循环

```
A → B → A → B → 死循环
```

### 保护措施
1. **深度限制**：触发链深度超过阈值则停止
2. **帧内唯一触发**：同一帧同一技能只触发一次
