# 猫三国 - 技能编辑器（策划工具）设计文档

> 最后更新：2026-04-15（根据实际代码同步）

## 一、项目目标

### 1.1 工具定位

技能编辑器是面向策划人员的核心生产工具，用于：

- 配置技能逻辑（无需写代码）
- 构建技能触发链（Trigger → Condition → Effect）
- 管理技能数据并导出到游戏运行时

### 1.2 当前状态

当前采用 **JSON直接编辑** 方式配置技能，尚未实现可视化编辑器。技能数据通过 `skills.json` 配置，由 `DataLoader` 加载，`SkillTriggerSystem` 运行时执行。

## 二、系统总体架构

### 2.1 当前架构

```
skills.json（手工编辑）
     ↓
DataLoader.LoadList<SkillData>()
     ↓
SkillTriggerSystem（运行时执行触发链）
     ↓
EventBus + BuffSystem（联动执行）
```

### 2.2 目标架构（待实现）

```
SkillEditor（独立工具或Web编辑器）
 ├── UI层（表单/节点图编辑器）
 ├── 数据层（Skill配置）
 ├── 校验系统（Validation）
 ├── 导出系统（Export → JSON）

--------------------------------

Runtime（游戏运行时，已实现）
 ├── DataLoader（加载JSON）
 ├── SkillTriggerSystem（触发链执行）
 ├── BuffSystem（Buff管理）
 ├── AttributeSet（属性系统）
```

### 2.3 数据流

```
策划编辑技能（JSON / 编辑器）
   ↓
生成Skill配置（JSON格式）
   ↓
DataLoader 加载到游戏运行时
   ↓
SkillTriggerSystem 解析执行
```

## 四、编辑方式

```json

  {

    "triggers": [

  }

```

## 五、节点系统设计

### 节点分类

**Trigger节点（触发）**
- OnAttack、OnHit、OnKill、OnDamaged、OnBuffAdd、OnSkillCast

**Condition节点（条件）**
- Random（概率）、HpBelow/HpAbove（血量）、BuffExists（Buff存在）、IsCrit（暴击）

**Effect节点（效果）**
- AddBuff、Damage、Heal、ModifyAttr、RemoveBuff

### 节点链结构

```
Trigger → Condition[] → Effect[]
```

## 六、校验规则

配置必须满足：
1. 必须有至少一个Trigger
2. 不能有断链（Trigger必须连接到Effect）
3. 节点类型必须合法
4. 参数必须完整（buffId不能为空等）

## 十、结论

技能编辑器 = 可视化战斗逻辑编程工具

当前通过JSON直接配置技能数据，运行时由SkillTriggerSystem执行。已具备完整的触发链执行能力。后续优先实现表单式Web编辑器，降低策划配置门槛。
