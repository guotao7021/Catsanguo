技能编辑器（策划工具）设计文档（产品级）
一、项目目标
1.1 工具定位

技能编辑器是面向策划人员的核心生产工具，用于：

可视化配置技能逻辑（无需写代码）
构建复杂技能触发链（Trigger → Condition → Effect）
管理技能数据并导出到游戏运行时
1.2 核心目标
✅ 零代码配置技能
✅ 可视化技能触发链
✅ 实时预览技能效果
✅ 与Buff / Attribute系统无缝联动
✅ 支持热更新与版本管理
二、系统总体架构
2.1 架构分层
SkillEditor（Unity Editor Tool）
 ├── UI层（Graph编辑器）
 ├── 数据层（Skill配置）
 ├── 校验系统（Validation）
 ├── 导出系统（Export）
 ├── 预览系统（Preview）

--------------------------------

Runtime（游戏运行时）
 ├── SkillSystem（触发链执行）
 ├── BuffSystem
 ├── AttributeSystem
2.2 数据流
策划编辑技能
   ↓
生成Skill配置（JSON）
   ↓
导入游戏运行时
   ↓
SkillSystem解析执行
三、核心功能设计
3.1 技能编辑方式（两种模式）
模式一：表单模式（初级策划）
事件：OnAttack
条件：30%概率
效果：添加Buff（中毒）

👉 优点：简单、快速

模式二：节点图模式（高级策划）⭐核心
[OnAttack]
     ↓
[Random 30%]
     ↓
[AddBuff]

👉 优点：

可视化
支持复杂链路
可扩展
四、节点系统设计（核心）
4.1 节点分类
1️⃣ Trigger节点（触发）
OnAttack
OnHit
OnKill
OnDamaged
2️⃣ Condition节点（条件）
概率（Random）
血量判断（HP）
距离判断
Buff存在
3️⃣ Effect节点（效果）
AddBuff
Damage
Heal
ModifyAttr
SpawnUnit
4.2 节点结构
class SkillNode
{
    public int id;
    public NodeType type;

    public List<int> nextNodes;
    public Dictionary<string, object> parameters;
}
五、技能数据结构（运行时）
JSON结构
{
  "id": 1001,
  "nodes": [

    { "id": 3, "type": "Effect_AddBuff", "buffId": 2001 }
  ]
}
