# 猫三国 - 战斗UI系统设计文档

> 最后更新：2026-04-15（根据需求文档同步）

## 一、系统定位

### 一句话定义

战斗UI = 玩家理解战局 + 做出决策的唯一窗口

### 核心目标
- 清晰展示战场信息（状态、血量、Buff）
- 支持操作（技能释放）
- 提供即时反馈（伤害数字、状态提示）
- 兼容自动战斗 / 手动技能操作

## 二、整体架构（已实现）

```
BattleScene (战斗场景，1142行)
 ├── DrawSquad()      → 单位渲染（位置/动画/血条）
 ├── DrawDeployUI()   → 部署阶段UI
 ├── DrawCountdown()  → 倒计时UI
 ├── DrawHUD()        → 战斗主界面
 ├── FloatingTextManager  → 飘字系统
 ├── RetreatNotificationManager → 撤退通知（武将不死亡）
 ├── CaptureResultPanel  → 俘获结算面板（新增）
 ├── SkillVFX         → 技能视觉效果
 └── ResultPanel      → 结算面板（含俘获信息）
```

### UI渲染方式

采用 **即时模式渲染**（Immediate Mode），通过SpriteBatch直接绘制：
- `SpriteBatch.Draw()` 绘制纹理/矩形
- `SpriteBatch.DrawString()` 绘制文字
- 无第三方UI框架，所有UI手动绘制
- 屏幕空间渲染（不受Camera2D影响）

## 四、核心模块设计

### 4.1 HUD（战斗主界面）

**显示内容**
- 战斗阶段标识（Deploy/Countdown/Fighting/Result）
- 玩家方信息
- 敌方信息

### 4.2 单位UI（Squad渲染）

**DrawSquad()渲染内容**
| 元素 | 说明 |
|------|------|
| 精灵 | Squad位置处绘制武将精灵 |
| 血条 | HP百分比条（红/绿色） |
| 名字 | 武将名称文字 |

### 4.3 技能UI

**玩家手动操作**
- 战斗中玩家可手动点击释放技能
- `UpdatePlayerSkillInput()` 检测玩家技能输入
- `TryUseSkill(squad)` 执行技能释放

**技能释放流程**
```
玩家操作 → 检查CD → GetPlayerSkillTargets() → 执行技能 → AddSkillVFX()
```

### 4.4 飘字系统（FloatingTextManager，已实现）

**飘字类型**

| 类型 | 颜色 | 说明 |
|------|------|------|
| 普通伤害 | 白色 | 常规攻击伤害值 |
| 暴击伤害 | 红色 | 暴击伤害（更大字体） |
| 治疗 | 绿色 | 恢复量 |

**实现方式**
- 创建浮动文字实例
- 向上漂浮 + 渐隐
- 自动管理生命周期

### 4.5 撤退通知（RetreatNotificationManager）

> **重要变更**：武将不会在战斗中死亡，改为"撤退"。

- 武将HP归零时显示撤退通知（非阵亡）
- 包含武将名称和撤退信息
- 自动消失

### 4.6 俘获结算面板（CaptureResultPanel，新增）

战斗结束后显示俘获结果：

```
┌─ 俘获结算 ────────────────────┐
│ 战斗胜利！                     │
│                                │
│ 俘获武将：                     │
│  [张飞] - 武力97 统帅85       │
│  [赵云] - 逃脱                │
│                                │
│ 操作：                         │
│  [招降] [释放] [关押]          │
└────────────────────────────────┘
```

- 每个被俘武将显示基本信息和6维属性
- 提供三种处理选项：招降/释放/关押
- 招降消耗资源，受忠诚度影响

### 4.7 技能VFX（SkillVFX，已实现）

**已实现效果**
- 技能释放时的扩展环形特效
- `AddSkillVFX(caster)` 在施法者位置创建VFX

```csharp
class SkillVFX {
    // 环形扩展效果
    // 随时间扩大半径 + 渐隐
}
```

### 4.8 结算UI（ResultPanel）

**显示内容**
- 胜利 / 失败标识
- 战斗统计（杀敌数/伤害量/技能使用）
- 奖励列表（XP/技能点）
- 俘获结果摘要（俘获了哪些武将）

**实现**：`ShowResultPanel()` + `AwardBattleRewards()` + `CaptureManager.ProcessCapture()`

## 六、与系统联动

| 联动系统 | UI表现 |
|----------|--------|
| 战斗引擎 | 实时更新Squad位置和状态 |
| EventBus | 战斗事件触发飘字/VFX/震屏 |
| BuffSystem | Buff变化触发图标更新 |
| SkillTriggerSystem | 技能触发触发VFX |
| MoraleSystem | 士气变化影响表现 |
| CaptureManager | 战斗结束后俘获判定→俘获面板 |

## 七、性能考量

- 即时模式渲染开销低（无UI框架overhead）
- SpriteBatch批量绘制
- 飘字/VFX有生命周期自动回收
- 2D渲染无GPU瓶颈

## 九、最终总结

战斗UI = 玩家决策系统

当前采用即时模式渲染，通过SpriteBatch直接绘制所有战斗UI元素。已实现飘字系统、技能VFX、屏幕震动和结算面板。术语从"阵亡"改为"撤退"以匹配武将不死亡机制，新增俘获结算面板用于战后武将处理（招降/释放/关押）。
